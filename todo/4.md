# 4 - Epic: FizzBuzz as a Service

# Description

Create a FizzBuzz service. Users will upload a file full of numbers to an API. The API will store the file in S3, and a worker service will transform the file into a FizzZBuzz file, and store it in a Postgres database.

Requirements:
- As this is a training exercise to keep skills sharp during unemployment, certain technologies will be required
- Services / Technolgies to be used:
  - S3 (Localstack)
  - SQS (Localstack)
  - Lambda (Localstack)
  - Postgres
  - Terraform
  - Kubernetes (k3s) - Note: the kubernetes cluster already exists
  - Skaffold
  - ArgoCD
- The files should be encrypted at rest
- Must have unit tests
- Must have integration tests
- Must have smoke tests
- Must have load tests
- Don't forget observability

The FizzBuzz algorithm to implement:
- for multiples of three, replace with "Fizz"
- for multiples of five, replace with "Buzz"
- for multiples of both three and five, replace with "FizzBuzz"
